## THE CORE PROBLEM

Every downloader on the internet eventually dies. Why?

```
Platform detects:
  1. Datacenter ASN (AWS/Render/Hetzner IP range) → block
  2. High request rate from same IP → rate limit / 429
  3. Missing browser headers/fingerprint → 403
  4. Missing auth tokens (YouTube PO token, Instagram session) → 401
  5. Known scraper User-Agents → 403

Traditional scrapers fail because they run from datacenter IPs
and look nothing like a real browser. Platforms have been training
ML models on this traffic for years.
```

**The solution is not fighting the platform. It's becoming invisible.**

## PLATFORM #1 — YouTube

### What YouTube Actually Checks

```
Every request to YouTube is analyzed for:

1. ASN (Autonomous System Number)
   → AWS = AS16509, blocked immediately for video fetching
   → Hetzner = AS24940, partially blocked for /watch, fully for streams
   → Residential ISP = AS55836 (Reliance Jio), AS24560 (Airtel) → allowed

2. Proof of Origin (PO) Token
   → Required since late 2023 for video stream URLs
   → Generated by JavaScript in the browser (a 1340+ char token)
   → Tied to visitor_data cookie (your browsing session)
   → Expires in ~6 hours

3. SABR/Botguard Challenge
   → JavaScript challenge that runs in the browser
   → Generates a challenge response that must accompany video requests
   → Can't be faked without running the JS

4. innertube API
   → All modern YouTube fetching uses this private API
   → Keys rotate but remain stable for months
   → Requires correct client context (ANDROID / iOS / WEB_EMBEDDED)
```

### Our Bypass Architecture

```
LEVEL 1 — Mirror Rotation (Easiest, 90% success)
───────────────────────────────────────────────
Invidious/Piped community instances are already authenticated
to YouTube as regular users with residential IPs. We just
query their public APIs instead of YouTube directly.

Endpoints:
  GET https://{piped-instance}/streams/{videoId}
  GET https://{invidious-instance}/api/v1/videos/{videoId}

200+ active instances worldwide. Rotate on failure.
Cost: ₹0. Reliability: 70-85% daily.
Risk: Instances go down/rate-limit often.

LEVEL 2 — Client-Side PO Token Injection (Medium, 99% success)
───────────────────────────────────────────────────────────────
Move the dangerous part to the USER'S browser.

Flow:
  1. Server generates an innertube API request (headers, body)
  2. Server sends this to the browser as a "fetchTask"
  3. Browser's JavaScript executes the fetch FROM a residential IP
  4. Browser gets the video stream URLs (with valid PO token context)
  5. Browser sends URL/metadata back to server
  6. Server uses the data for further processing

Why this works:
  The request comes from a residential IP (user's home/phone)
  YouTube's bot detection sees a real browser, real IP
  No PO token needed — the user IS the PO token generator

Implementation:
  Server → POST /api/dl/prefetch → returns { task: {...} }
  Browser JS → executes fetch(task.url, task.options)
  Browser JS → POST /api/dl/submit-result → sends metadata
  Server → processes and returns stream options to UI

LEVEL 3 — OAuth2 Headless Bypass (Advanced, 99.9% success)
────────────────────────────────────────────────────────────
Use yt-dlp's OAuth2 plugin on the VPS.

How it works:
  1. User visits a one-time URL, logs in with their Google account
  2. We get an OAuth2 refresh_token (stored server-side, encrypted)
  3. Every download uses this token → YouTube sees a signed-in user
  4. Download requests look identical to YouTube Premium user
  5. No PO token needed — OAuth2 session supersedes it

Implementation:
  pip install yt-dlp[oauth2]
  yt-dlp --username oauth2 --password "" URL

The refresh_token lasts months. Renews automatically.
Risk: If Google bans the account. Mitigate: use throwaway accounts.
```

## PLATFORM #2 — Instagram / Reels / Stories

### What Instagram Checks

```
1. Session Cookie (ds_user_id + sessionid)
   → Required for ANY private content
   → Even public Reels now require a soft session check
   → Scrapers without cookies get 401 or redirect to login

2. X-IG-App-ID header
   → Must match a valid Instagram app ID
   → Main web app: 936619743392459
   → Instagram Android: 567067343352427

3. CSRF Token
   → Required for mutating requests
   → Extracted from cookies (csrftoken field)

4. Rate Limiting per IP
   → 200 requests/hour per IP for embed endpoints
   → Datacenter IPs get much lower limits (~20/hour)

5. Content-Type Fingerprint
   → Instagram checks if the browser sent proper Accept headers
   → Scraper user-agents result in 403
```

### Our Bypass Architecture

```
LEVEL 1 — oEmbed API (Simplest, public content only)
──────────────────────────────────────────────────────
Instagram has a public oEmbed API:
  GET https://graph.facebook.com/v18.0/instagram_oembed
      ?url=https://www.instagram.com/p/{shortcode}/
      &access_token={APP_ID}|{CLIENT_TOKEN}

Works for: Public posts and Reels
Doesn't work: Stories, private accounts, highlights
Facebook developer account needed (free, permanent)

LEVEL 2 — Proxy Rotation + Header Spoofing (Medium)
─────────────────────────────────────────────────────
Use our 100,000+ proxy pool with rotated residential IPs.
Add correct headers to mimic mobile Safari:

headers = {
  "User-Agent": "Mozilla/5.0 (iPhone; CPU iPhone OS 17_0 like Mac OS X)...",
  "X-IG-App-ID": "936619743392459",
  "Accept": "text/html,application/xhtml+xml...",
  "Accept-Language": "en-US,en;q=0.9",
  "Referer": "https://www.instagram.com/",
}

Success rate: 70-80% with fresh residential proxies
Falls through: Proxy detection, soft bans after ~50 requests/IP

LEVEL 3 — Puppeteer Stealth (Nuclear Option, 99% success)
───────────────────────────────────────────────────────────
Run headless Chrome on VPS with puppeteer-extra-plugin-stealth.

What stealth plugin does:
  - Randomizes navigator.webdriver (hides automation flag)
  - Spoofs Canvas fingerprint
  - Randomizes screen resolution, timezone, language
  - Fakes hardware specs (CPU cores, RAM, GPU)
  - Passes ALL bot detection tests (Cloudflare, DataDome, PerimeterX)

Cost: ~150MB RAM per Chrome instance on VPS
Limit: Run 2-3 concurrent Puppeteer sessions on CX22 (4GB RAM)
Use case: Rare fallback for when all other methods fail

LEVEL 4 — Instagram Direct API (Private content with user OAuth)
─────────────────────────────────────────────────────────────────
Instagram Basic Display API + Instagram Graph API (official)
User logs in via OAuth → we get access_token → can download THEIR content

Endpoint: GET /me/media?fields=media_url,media_type,thumbnail_url
This is ALWAYS allowed by Instagram's ToS for user's own content.
```

## PLATFORM #4 — TikTok

### What TikTok Checks

```
1. device_id + install_id (generated per app install)
2. X-Khronos timestamp + X-Gorgon HMAC signature
   → Every request must be signed with a rotating secret key
   → Key extracted via reverse engineering of the Android APK
   → Current key: changes every 3-6 months
3. Watermark removal requires TikTok's API key (paid tier)
```

### Our Bypass

```
LEVEL 1 — TikTok's own embed API
  GET https://www.tiktok.com/oembed?url={videoUrl}
  Returns metadata including thumbnail. BUT no direct video URL.

LEVEL 2 — Third-party CDN nodes
  TikTok serves videos from CDN nodes:
  v19-webapp.tiktok.com, v19-us.tiktok.com
  
  Video ID extraction: parse from /video/{videoId} page HTML
  Then build CDN URL directly (no auth needed if you have the video ID)

LEVEL 3 — Cobalt.tools (best option)
  Handles TikTok without watermark
  Self-host → unlimited
  
LEVEL 4 — Snaptik / tikwm.com API
  Free third-party APIs specifically for TikTok
  GET https://tikwm.com/api/?url={tiktokUrl}
  Returns: play (with watermark), wmplay (without watermark), music, author
  Free tier: 100 req/day per IP. Rotate IPs → effectively unlimited.
```

## THE IP INVISIBILITY STACK

This is the answer to "how we bypass every wall."

```
RESIDENTIAL IP SOURCES (free, always rotating):

1. User's Own Browser (best)
   → Move the fetch to CLIENT-SIDE JavaScript
   → Request originates from user's residential IP
   → Platform sees a real home internet user
   → Zero cost, zero detection, zero rate limit

2. Free Proxy Pool (100k+ daily)
   → ProxyScrape, GeoNode, ProxyNova, GitHub lists
   → Already running in our ProxyScraper service
   → Rotate per request → each request from different IP
   → 60-70% success rate per proxy, but at 100k pool = reliable overall

3. Tor Network (anonymous, free)
   → SOCKS5: 127.0.0.1:9050 (install tor on VPS)
   → Residential-adjacent: each exit node is a different IP
   → Slower (~2s latency) but completely free and permanent
   → `sudo apt install tor && systemctl start tor`

4. Cloudflare Workers (edge IPs, not datacenter)
   → CF runs on 300+ cities, all on clean IP ranges
   → $0 for 100k requests/day
   → Make the dangerous fetch FROM CF Worker → clean IP

5. OAuth2 / Real Sessions
   → Doesn't matter what IP, you're authenticated as a user
   → Platform can't block without banning the account
   → YouTube, Instagram, TikTok all honor valid sessions
```
